# SignalStack — Anchor Principles

- Density over decoration: the first screen is the actual incident workspace, not a marketing hero, product mockup, or mascot.
- Neutral canvas, restrained signal accent, status chips for the only color that carries meaning.
- Table-first hierarchy: signals, source, owner, state, and recent activity are scannable before prose.
- Selecting a row reveals context and actions without destroying queue position or requiring a page transition.
- Workflow stages and permission boundaries make ownership explicit; generic feature cards do not.
- Loading, empty, error, and success states are designed as first-class operating moments.
- Every figure is a bracketed placeholder; no invented metrics, prices, customers, or uptime claims.
- Responsive layouts keep filters, table context, detail state, and action controls usable instead of collapsing into an unreadable desktop screenshot.
