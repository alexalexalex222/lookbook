Getting started

Quickstart

Install the CLI, describe your data, and get a typed, reactive database running on your machine. About five minutes from here to a query that updates itself.

#Install

Epoch is one package and one CLI. Add it to an existing project, or scaffold a new one with the starter — either way you'll have a local database, schema push, and type generation wired up.

Starting fresh? The init command sets up the project structure and an example schema for you.

#Describe your data

A schema is plain TypeScript. Define your tables, their columns, and the indexes you'll query by. This file is the source of truth for the whole stack — your client types are generated from it.

#Run the dev server

Start Epoch in watch mode. It runs a local database, pushes schema changes as you save, and regenerates the typed client so your editor always knows the shape of your data.

Leave it running
Every save re-pushes your schema and re-runs codegen in milliseconds. You should never have to restart it or generate types by hand.

#Write a query and a mutation

Reads and writes live in functions next to your data. Queries are reactive by default; mutations run inside a serializable transaction. Validation and authorization belong here, on the server.

#Subscribe from your app

Wrap your app once with the provider, then call your functions through the generated api. useQuery opens a live subscription — it returns the current result, then every result after it.

That's the whole loop. Call send from anywhere — another tab, another user, a background job — and every mounted Channel updates in the same instant.

#Deploy or self-host

When you're ready, ship to Epoch Cloud with one command, or run the same build yourself. Epoch is a single binary with no external dependencies — your database, the function runtime, and the changefeed in one process.

No lock-in
The wire protocol and the server are open source under the MIT license. Self-hosting is a first-class path, not a downgrade — the cloud just runs the same binary for you.
